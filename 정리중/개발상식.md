# 1. Semantic Version

```
MAJOR.MINOR.PATCH
```

- 소프트웨어 버전 번호를 관리하기 위한 규칙 세트
- 프로젝트 간 의존성을 효과적으로 관리

- `MAJOR` : 업데이트 시 역호환되지 않는 변경사항이 포함될 경우 증가
- `MINOR` : 역호환성을 유지하며 기능 추가시 증가
- `PATCH` : 역호환성을 유지하며 버그 수정시 증가
